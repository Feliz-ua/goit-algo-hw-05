## Висновки
### Текст 1:
* Для існуючого підрядка 'Java' найшвидшим є алгоритм Боєра–Мура (0.187574 s), KMP приблизно в 2.9 раза повільніший, а Рабіна–Карпа — у 4.4 раза повільніший за Боєра–Мура.

* Для вигаданого підрядка 'abcdef1' Боєра–Мура знову найшвидший (0.438417 s), KMP повільніший приблизно у 5.6 раза, а Рабіна–Карпа — у 8.5 раза.

Алгоритм Боєра–Мура стабільно найшвидший як для знайденого, так і для відсутнього підрядка. KMP - другий, Рабіна–Карпа — найповільніший.

### Текст 2:
* Для існуючого підрядка 'Java' Боєра–Мура знову найшвидший (1.085591 s), тоді як KMP приблизно в 3.2 раза повільніший, а Рабіна–Карпа — у 4.9 раза повільніший.

* Для вигаданого підрядка 'abcdef1' Боєра–Мура показує найменший час (0.618055 s), KMP повільніший приблизно у 5.6 раза, Рабіна–Карпа — у 8.7 раза.

Результат такі самі: Боєра–Мура — найшвидший, KMP — середній, Рабіна–Карпа — повільний для обох типів підрядків.

# Загальний висновок по всіх тестуваннях:
* Алгоритм Боєра–Мура виявився найшвидшим у всіх чотирьох сценаріях (2 тексти × 2 підрядки). Його ефективність особливо помітна для довших текстів, завдяки можливості «перестрибувати» кілька символів за одну ітерацію.

* Алгоритм Кнута–Морріса–Пратта (KMP) стабільно займає друге місце: через додаткову обробку програє Боєра–Мура на реальних даних.

* Алгоритм Рабіна–Карпа в даних умовах виявився найповільнішим.
